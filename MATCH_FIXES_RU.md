# FRAKTUM — исправления матча

## Что было исправлено

1. **Зависание ИИ при `BLIND_TOP`.**
   - ИИ теперь рассматривает верхнюю карту колоды, а не карты руки.
   - Движок принимает только текущую верхнюю карту колоды.
   - После отклонённого действия ИИ безопасно завершает ход.
   - В интерфейс добавлена кнопка действия прямо на колоде: `PLAY TOP`.
   - Добавлена дополнительная защита от четырёх одинаковых неразрешившихся действий ИИ подряд.

2. **Двойное завершение боя и двойное начисление победы.**
   - `finishBattle` и `checkWinCondition` сделаны идемпотентными.
   - После перехода в `betweenBattles` обработчик хода больше не проверяет победителя повторно.

3. **Ошибочный урон `-3 HP` при обычном завершении хода.**
   - Усталость применяется только тогда, когда одновременно пусты и рука, и колода.
   - Источник урона явно записывается как `Deck exhaustion`.

4. **Ложные предупреждения отладчика об уроне без источника.**
   - Исправлено чтение ограниченного журнала из 80 строк: новые строки определяются по пересечению старого и нового журнала.
   - Уменьшение руки больше не помечается как `draw`.
   - Скрытые зоны противника больше не создают ложные переходы `card -> unknown`.
   - При создании нового матча сбрасываются счётчики структурированных событий.

5. **Неполная работа рулетки судьбы.**
   - `MERGED_DECKS` теперь действительно объединяет оставшиеся колоды, перемешивает их и делит заново.
   - Эффект рулетки завершается перед следующим личным ходом владельца и не остаётся навсегда.
   - `HIDDEN_HAND` визуально скрывает карты в руке, но позволяет разыграть их вслепую.

6. **Нестабильные служебные события.**
   - Идентификаторы тактических раскрытий теперь детерминированы и не используют `Date.now()` внутри движка.
   - Исправлены локальные ошибки линтера в компонентах матча: D20, полосы HP/Will и журнал боя.

## Добавленные проверки

- верхняя карта действительно разыгрывается при `BLIND_TOP`;
- карты руки блокируются при `BLIND_TOP`;
- ИИ завершает ход без бесконечного цикла;
- усталость не срабатывает при наличии карт;
- смерть от усталости начисляет только одну победу за бой;
- повторная проверка победы не меняет счёт;
- `MERGED_DECKS` сохраняет все карты;
- рулетка корректно истекает;
- ограниченный журнал отладчика не теряет новые строки;
- стресс-тест из 50 детерминированных боёв завершается без тупика.


- TypeScript и production-сборка Vite: успешно.

## Важно

Команда `npm run lint` проверяет вообще весь проект. В исходном архиве остаются старые lint-ошибки вне матча — в Pack Opening, Hub, Market и некоторых общих сервисах. Они не относятся к исправлениям матча и не блокируют `npm run build`. Для проверки именно матча используется `npm run lint:match`.

# Исправление быстрого перехода по хабу и Supabase Realtime

## Исправленный спавн

Быстрые кнопки комнат больше не используют произвольные `spawnPoint` карты.
Теперь они вычисляют точку появления по тем же `spawnOnTarget`, которые применяются при ручном переходе через выход.

Точки входа из центрального хаба:

- Market: `{ x: 764.4, y: 98.9 }`
- Archive: `{ x: 454, y: 1360 }`
- Arena: `{ x: 900, y: 620 }`

При возврате кнопкой Hub используется точка выхода конкретной комнаты:

- Market -> Hub: `{ x: 650, y: 960 }`
- Archive -> Hub: `{ x: 768, y: 220 }`
- Arena -> Hub: `{ x: 100, y: 500 }`

При переходе напрямую между боковыми комнатами используется их обычный вход из центрального хаба.
Общие стартовые точки комнат также заменены на безопасные входные точки, поэтому обновление страницы внутри комнаты не помещает персонажа в коллайдер.

## Исправление Realtime

Сообщение:

`Realtime send() is automatically falling back to REST API`

возникало, когда отправка движения выполнялась в момент, когда Realtime-канал уже считался подписанным кодом приложения, но WebSocket временно не находился в состоянии `joined`.

Исправления:

- broadcast отправляется только при `channel.state === "joined"`;
- устаревший канал после переподключения не может отправлять координаты;
- автоматический REST fallback для движения больше не вызывается;
- подтверждение каждого частого broadcast отключено (`ack: false`);
- при повторной подписке старый heartbeat очищается, поэтому интервалы не размножаются;
- состояние `CLOSED` корректно сбрасывает флаг подписки.

`httpSend()` намеренно не используется: координаты движения являются частыми временными данными и не должны отправляться через REST при каждом шаге.
